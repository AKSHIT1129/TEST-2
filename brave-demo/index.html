Threat Vector Matrix

An educational database documenting how common online frauds operate and how FraudShield mitigates them.

Account Takeover (ATO)

The Attack: Fraudsters purchase leaked passwords from the dark web and use automated bots to mass-login to accounts.

System Solution:
Engine calculates distance between user's last IP and current IP. If distance > 1000m in < 2 hrs, it enforces strict block.

Phishing & Smishing

The Attack: Attackers send SMS messages posing as banks, causing panic to trick users into giving away OTPs.

System Solution:
NLP scans text streams for urgency keywords, alerting the user before the malicious link is ever loaded.

Micro-Transaction Fraud

The Attack: Instead of stealing $5000 at once, fraudsters process 40 rapid charges of $1.00 (Card Testing) to evade baseline volume alerts.

System Solution:
The Velocity Time-Series engine clusters data over 5-minute intervals. 40 micro-charges instantly triggers lockdown.

Headless Botnets

The Attack: Organized crime groups use massive server farms running phantom web browsers to execute mass-fraud without human intervention.

System Solution:
The Device Sandbox inspects WebGL graphic renderers and mouse entropy to identify the structural differences between software and human.

Credential Stuffing Scripts

The Attack: A python script is programmed to inject thousands of stolen username/password combinations directly into the login form inputs instantly.

System Solution:
The Biometrics ML model analyzes typing flight time. If a 15-character password is typed in exactly 0.001 seconds with 0 errors, the token is voided.
